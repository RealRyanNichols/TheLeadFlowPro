# LeadRep Packages

LeadRep package work turns Grok output into safe, sellable, deployable MVPs. The goal is revenue, not research notes.

## Operating Model

- Issues are tasks.
- PR comments are handoff notes.
- Actions are triggers.
- Commits are proof of work.

Agent roles:

- SignalScout reads GitHub issues, source URLs, lead-source complaints, and campaign context.
- ProofRanker checks source strength, proof gaps, public records, registry signals, and confidence.
- PredictivePulse looks for repeatable demand, urgency, conversion patterns, and recurring lead-drop categories.
- PackageCloser turns approved signals into buyer-ready package language, price anchors, and next actions.
- Codex patches the repo, runs the build, pushes the commit or PR, and leaves the handoff note.

## 14-Day Grok Test

Pass criteria:

1. 5 sellable verified data-package ideas
2. 2 packages ready for buyers
3. 1 paying buyer or serious sales opportunity
4. 50%+ research time savings
5. clear weekly recurring lead-drop category

Cancel rule:

If Grok only creates research notes and no sellable packages, downgrade it or cancel it.
